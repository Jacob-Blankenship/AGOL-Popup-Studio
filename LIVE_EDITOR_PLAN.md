# Live Web Map Editor — Pull & Push Popup Plan

## The Short Answer

**Yes — completely possible, and less painful than you might think.** The ArcGIS REST API fully supports reading and writing web map data from a browser app. The main requirement is a one-time **app registration** in AGOL to get an OAuth Client ID. After that, users sign in directly inside the Studio — no Python, no copy-pasting.

---

## How Authentication Works

> IMPORTANT:
> The recommended approach is **OAuth 2.0** — the same "Sign in with AGOL" flow you already use on Esri products. This is the **only** way to read private web maps from a browser safely. You register the Studio as a trusted app once, and after that any AGOL user in your org can sign in with a single click — no passwords stored anywhere in the code.

The alternative (generateToken with username + password in the browser) works but is **less secure** and increasingly deprecated by Esri. We'd avoid it.

---

## Proposed Workflow

```
1. User clicks "Sign In to AGOL"  ->  standard AGOL OAuth login popup
2. Token returned to Studio  ->  user is authenticated, username shown in header
3. User enters Web Map ID  ->  clicks "Load Map"
4. Studio fetches web map JSON  ->  ALL layers discovered automatically
5. Each layer becomes its own TAB in the Live Simulation panel header
6. User clicks a layer tab  ->  sidebar config (fields, colors, font, etc.) switches to that layer's unique state
7. User pulls that layer's current popup from AGOL  ->  existing config loaded into the editor
8. User makes edits independently for that layer
9. User clicks "Push Layer"  ->  only that layer's popup is written back to AGOL
10. User switches to another tab and repeats independently
```

## Import Path A — Paste HTML (No Auth Required)

This is the **simplest and fastest** way to resume editing a previous run. Since every popup
generated by AGOL Popup Studio contains all its configuration baked into inline CSS and HTML
structure, pasting the popup HTML back into the Studio fully restores the editor state.

### How it works

```
1. User runs the Python script  ->  popup is pushed to their AGOL web map
2. User opens the popup in AGOL  ->  copies the raw HTML source
   (or copies directly from the Python script output)
3. User comes back to AGOL Popup Studio
4. Pastes the HTML into the "Import from Previous Run" panel
5. Clicks "Import"  ->  Studio parses and restores full editor state
6. User makes their tweaks
7. Downloads the updated Python script and runs again
```

### What gets parsed from the HTML

| Data                  | Source in HTML                                             |
|-----------------------|------------------------------------------------------------|
| All popup colors      | Inline style= attributes on every element                  |
| Field aliases         | Left <td> text content                                     |
| Field names           | Right <td> tokens ({field_name} / {expression/fmt_*})      |
| Field order           | Row order in <tbody>                                       |
| URL fields            | Fields rendered as <a href=...> vs plain text              |
| Button text           | Text content of the <a> tag                                |
| Hidden fields         | Absent from HTML - inferred                                |
| Font family           | font-family in table/header style                          |
| Header alignment      | text-align in header div style                             |
| Popup header template | Header div text content (e.g. {nickname})                  |

NOTE: The <!-- AGOL_POPUP_STUDIO --> marker is used to validate the pasted HTML before parsing.
If it's missing, the Studio warns the user that the popup wasn't generated by this tool
and color recovery may be incomplete.

### New Component: src/components/HtmlImporter.jsx
- A collapsible panel in the sidebar labeled "Import from Previous Run"
- Contains a <textarea> for pasting HTML
- An "Import" button that runs the parser and hydrates state
- Status line: e.g. "Imported 12 fields from previous popup" or "No Studio marker found — fields imported, colors defaulted"
- Works completely offline — no network calls, no auth

### New Utility: src/utils/htmlParser.js
- Pure JavaScript HTML parser using the browser's built-in DOMParser
- Extracts colors from inline styles via regex/attribute walking
- Maps <tr> rows into { fieldName, alias, isUrl, buttonText } objects
- Returns a full config object ready to load directly into App state

## Import Path B — Live AGOL OAuth (Full Round-Trip)

---

### New Component

#### [NEW] src/components/LiveEditor.jsx
A new collapsible panel in the sidebar with three phases:

**Phase 1 — Auth Strip**
- A clean "Sign In to AGOL" button using UserSession.beginOAuth2()
- Shows the signed-in username + org once authenticated
- "Sign Out" option

**Phase 2 — Map Loader**
- User enters (or auto-fills from existing config) a Web Map ID
- Click "Load Map" -> fetches full web map JSON
- All discoverable layers are extracted and stored in a layers[] array in state
- Each layer holds its own independent config object:
  { schema, fieldOrder, fieldAliases, hideFields, urlFields, urlButtonTexts, colors, popupHeader, popupAlign, popupFontFamily }

**Phase 3 — Per-Layer Push Controls**
- Each layer tab has its own "Pull" and "Push" button
- Pull: Reads that layer's existing popupInfo, parses the HTML, and hydrates only that layer's config state
- Push: Writes only that layer's updated popup back to the web map via updateItem() — other layers are untouched
- Status feedback shown inline per-layer (success/error badge on each tab)

---

### [NEW] Multi-Layer Tab System in LivePreview

The **Live Simulation panel** gets a tab bar across its header:

```
+-------------------------------------------------------------+
|  [Tax Parcels x]  [Road Centerlines x]  [Zoning Districts x]|  <- tabs
+-------------------------------------------------------------+
|                                                             |
|            Live popup preview for active tab layer          |
|                                                             |
+-------------------------------------------------------------+
```

- Clicking a tab switches both the Live Preview AND the entire sidebar config (field order, colors, etc.) to that layer's state
- Each tab shows the layer's name, pulled directly from the web map JSON
- Tabs only appear after "Load Map" is clicked — in the existing single-layer mode the interface stays exactly as it is today
- An "x" on each tab removes that layer from the session (does not delete it from AGOL)
- A "+" button on the tab bar allows manually adding a layer by name (for cases where schema import is needed without OAuth)

## Authentication Setup (One-Time, Admin Only)

> NOTE: This only needs to be done **once by you (or an admin)**. End users don't need to do anything special.

1. Go to AGOL -> Content -> My Apps -> Register New App
2. Set Redirect URI to: https://jacob-blankenship.github.io/AGOL-Popup-Studio/
   (and http://localhost:5173 for local dev)
3. Copy the generated Client ID and hard-code it into the app config

---

## What "Pull" Parses

The existing popup HTML generated by Studio is tagged with <!-- AGOL_POPUP_STUDIO -->. When pulling:

- If the tag is found -> cleanly extract and re-hydrate Studio state (colors, field order, aliases)
- If the tag is NOT found (popup was made manually in AGOL) -> import fields/aliases only; colors default to the current template

---

## What Gets Pushed

The push operation sends the same JSON the Python script currently generates, but via updateItem() directly — no Python required.
The <!-- AGOL_POPUP_STUDIO --> marker is preserved so future pulls still work correctly.

## Risk & Limitations

| Concern                | Reality                                                                    |
|------------------------|----------------------------------------------------------------------------|
| Auth complexity        | Low — OAuth handled entirely by Esri's arcgis-rest-js library              |
| CORS issues            | None — AGOL REST API is explicitly CORS-enabled for registered apps        |
| Permission errors      | User must own or have edit access to the web map                           |
| Manually-built popups  | Pull will import fields only; colors won't be inferred                     |
| Token expiry           | arcgis-rest-js handles automatic token refresh                             |
| Multi-layer state size | Each layer stores its own full config; large maps (20+ layers) may need pagination |
| Schema conflicts       | If two layers share field names, each is still isolated — no cross-contamination |

## Open Questions

1. IMPORTANT: Do you have access to register apps in your AGOL organization?
   You'll need the Client ID from a registered app to enable OAuth.
   If you're an admin this takes ~2 minutes. If not, coordinate with whoever manages your AGOL org.

2. Should the Live Editor replace the Python script path or sit alongside it?
   Recommendation: keep both — Python is great for batch jobs across many maps,
   Live Editor is faster for single-map edits.
